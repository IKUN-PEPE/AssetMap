暴露面搜索需求与问题整理

一、暴露面搜索
1. Playwright 抓取搜索引擎页面可能不稳定，选择器、验证码、风控都要有兜底。
2. BackgroundTasks + Playwright 长任务容易阻塞或失控，需要并发限制、超时控制、状态回写。
3. “确认导入资产库”必须严格复用 `save_assets` 和去重逻辑，不能直接写 `WebEndpoint`。
4. GitHub、网盘、文档类链接不一定适合导入 `WebEndpoint`，应优先作为线索或证据。
5. 测试不能只停留在 `test_exposure_search.py`，还需要补 API、导入、Provider Mock、风控场景、前端构建、后端全量测试。
6. 数据模型新增表后，必须确认 migration、`init_db.py`、SQLite 测试环境、PostgreSQL 兼容性。
7. 不允许为了 Playwright 方便而绕过验证码、登录限制或自动下载文件。
8. 搜索引擎结果需要支持记录：例如搜索“深圳地铁 pdf 文件”时，允许用户自行选择记录，或在 Playwright 前台手动点击后记录结果。
9. 项目中不要直接写代码，先反问我并规划，再实施；使用 skills superpowers。
10. 继续未完成任务时，不要帮我做任何 Git 提交。

二、悬浮窗与交互
1. “AssetMap Discovery” 默认打开，并且一直保持在最上层。
2. `http://localhost:61800/` 无法访问时，需要排查或修复。
3. 使用 Playwright 自动处理验证码弹窗、登录限制，甚至复杂的 iframe 嵌套；使用 skills superpowers。
4. 点击“完成并继续”时，如果当前搜索语法已经完成，应立即关闭当前浏览器页并跳到下一条搜索语法，不要长时间停留。
5. 文档类链接点击后应直接预览，而不是下载：
   - PDF
   - DOC / DOCX
   - XLS / XLSX
   - CSV
   - SQL
   - JSON
6. 悬浮球点击“下一页”应真正推动页面跳转，而不是无效按钮或仅依赖历史记录。

三、结果同步与刷新
1. 当前任务运行时，需要显示：
   - 搜索进度条
   - 当前搜索语法
   - 下一条搜索语法
   - 所有搜索语法完成后直接结束任务
2. 暴露面搜索前端不要默认自动刷新。
3. 需要增加手动刷新功能：点击一次只刷新一次。
4. 需要增加自动刷新开关，并允许设置刷新间隔：
   - 5 秒
   - 10 秒
   - 15 秒
   - 30 秒
5. “搜索语法明细”和“结果列表”必须保持同步。
6. “总结果数”和“搜索语法明细里的结果数”必须一致。
7. 当前已确认的两个 bug：
   - 悬浮球点击“下一页”无效
   - 搜索语法明细与结果不同步

四、Git 约束
1. 本次会话中，如果需要 Git，只给我命令，不要帮我提交。
2. 分析整改项目产生的元数据、测试文件，以及哪些可以清理。

五、项目重构与环境
1. “选择集”功能的前后端源码、数据库表和关联逻辑全部删除，不再保留。
2. 重写数据库初始化文件以及 Docker / Makefile 文件，实现 Docker 一键启动数据库。
3. 需要补一个 `start-project.ps1`，一键执行：
   - 启动数据库
   - 初始化数据库
   - 启动后端
   - 启动前端
