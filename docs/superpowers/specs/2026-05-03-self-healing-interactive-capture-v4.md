# 自动增强型交互采集 (Self-Healing Interactive Capture) 设计方案

**日期：** 2026-05-03
**状态：** 已评审 (v4)

## 1. 目标
通过“自动检测-人工解封-一键恢复”的混合逻辑，彻底解决搜索引擎风控导致的抓取停滞或窗口“一闪而过”的问题。

## 2. 系统状态机逻辑

### 2.1 状态转移定义
- **NORMAL**: 自动执行 `query_selector` 提取结果。
- **RETRYING**: 检测到风控，清理 Cookie/缓存并刷新重试。
- **INTERVENTION_REQUIRED**: 连续 3 次失败，锁定任务，弹出窗口，等待用户信号。
- **RECOVERING**: 收到用户 `__am_resume_auto` 信号，重置计数器并尝试回执 NORMAL 提取。

## 3. 技术实现

### 3.1 注入脚本增强 (discovery_script.js)
- **风控检测**：监听 DOM 变化，若匹配到 "captcha", "verify", "login" 等关键词，UI 自动切换为“风控警告”模式。
- **恢复信号**：用户点击 `恢复自动抓取` 时，脚本调用后端 RPC。

### 3.2 通信协议
- **RPC 1**: `__am_record_clue(data)` - 手动记录单个线索。
- **RPC 2**: `__am_resume_auto()` - 通知后端重新开始自动扫描当前 DOM。

### 3.3 后端服务适配 (ExposureSearchService)
- 在 Provider 抓取主循环中增加 `retry_count` 逻辑。
- 使用 `asyncio.Event` 挂起协程，实现无损等待用户操作。

## 4. 容错与安全
- **超时保护**：人工干预模式最长等待 10 分钟，超时则跳过该 Provider。
- **并发锁**：进入干预模式时，强制将该任务的并行数降为 1，确保用户专注处理。

## 5. 验证计划
- **逻辑测试**：手动向页面注入风控 HTML，验证工具栏是否变色并显示恢复按钮。
- **闭环测试**：点击恢复按钮后，验证后端是否正确触发了新的提取动作。
